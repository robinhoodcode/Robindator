# Contributing

Contributions that improve accuracy, transparency, or safety are welcome.

## Documentation changes

1. Use primary sources whenever possible.
2. Link protocol claims to the relevant documentation or transaction.
3. Distinguish intended behavior from completed on-chain activity.
4. Never publish private keys, seed phrases, credentials, or personal data.
5. Keep financial claims factual and avoid promises of returns.

## Deployment records

Contract, pool, lock, and buyback entries must include a working block-explorer
link or transaction hash. Maintainers should independently verify addresses
before merging.

## Pull requests

Keep each pull request focused and explain:

- what changed;
- why it changed;
- how the claim was verified; and
- any remaining uncertainty or risk.

Generated files, vendored dependencies, and unrelated formatting changes should
not be included solely to inflate the size of a contribution.
