# Security policy

## Reporting a vulnerability

Do not open a public issue for an unpatched vulnerability that could put funds,
keys, contracts, or users at risk. Contact the repository owner privately
through the security-reporting channel listed on the GitHub profile.

Include:

- affected component and version;
- impact and prerequisites;
- reproducible steps or a proof of concept;
- relevant transaction hashes; and
- a suggested remediation, if known.

Never send a private key, seed phrase, or live credential.

## Scope

This repository does not currently contain a verified NOXA token deployment or
flywheel contract. The legacy wallet-adapter prototype under `src/` and `lib/`
should not be treated as audited production code.

Protocol-level issues in NOXA should be reported to the protocol maintainers
through their official channels. Include only public, non-sensitive details in
this repository.

## No bug bounty

No bug bounty is promised unless a separate, explicit program is published.
Responsible reports are still appreciated.
